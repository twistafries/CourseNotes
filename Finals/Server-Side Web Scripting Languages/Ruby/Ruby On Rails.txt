RUBY ON RAILS

-Created by David Heinemeier Hansson by extracting Ruby on rails from his work on
 his project management tool Basecamp
-Ruby on rails is not only free to use, it is an open source software that means
 that you can also help to make it better by contributing codes.
-Ruby on rails is an MVC Architecture which is divided into 3 interconnected parts
 which is the Model that handles the business logic, Controller that control the flow
 of the application, And then the View which handle the logic presentation that the end
 user sees.
-Ruby on rails is Convention over Configuration, that helps decrease the number of decisions that a
 developer attempt so that it will not lose flexibility in coding.
-DRY code is used in Ruby on rails DRY which stands for Don't Repeat Yourself that makes the code concise,
 consistent that makes it easier to maintain
-Ruby on rails is easy to learn most of the code are based in english language and it is an Object Oriented language
-Most of the top websites used Ruby on rails as server-side  web application by top websites like 
 twitter/shopify/sondcloud/slideshare/urbandictionary/bloomberg/github
-Using Ruby on rails can make the development faster. 

ADVANTAGES

	-It has a consistent structure it means it have a strong foundation and library
	-Can easily manage database migrations because it allows us to create and edit database column and row
	-Can easily do prototyping and can be used in rapid development
	-Can easily learned due to its good english word used
	-It is a multi-paradigm language and Object Oriented
	-Ruby gem is a package manager that extends the core functionality of the framework similar to npm node.js
	-Due to its Maturity ruby on rails is a respected Community



DISADVANTAGES
	-Ruby on rails have a big framework so it makes the performance slower
	-Lack of Documentation
	-On cutting age of development
